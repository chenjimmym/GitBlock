What is Git?

Git is a version control system; you can think of it as a series of snapshots(commits) of you code base. With the Creation of each snapshot you build out a whole application, seeing the growth of a full app. During this time you can make branches to epeeriment and comeabck to a previous snapshots you took or merge those experiments into your app.

What is GitHub?

GitHub is a cose hosting platform for verson control and collaboration. It lets you and others work together on projoects from anywhere using those cose snapshots that you took during development.

Git Init

Git init is the act of registering this current code base

Git add

Git add is how we stage the integration of our current code base into our current list of snapshots

Git Status

Git Status allows us to see the changes we are going to make

Git commit

Git commit; finally we can commit those changes we added to our last snapshot giving us a new snapshot of out code with whatever new chnages were made

Git remote add

Git remote add is used to set up a online repository where we can place all of our current code. This is where the outside world can see our code base.

Git Push

Git push is how we get our code to Gitbub, an excellent place to store all of our repositories. Other developers might look at our existing projects to see how we approached probles, pull down our code and add ndew features or even update our code base with features they wanted to add
